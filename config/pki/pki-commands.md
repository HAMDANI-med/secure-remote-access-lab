# PKI Reference — easy-rsa 3.x

All commands are run from `/etc/openvpn/easy-rsa/` as root (or with sudo).

---

## Initialize the PKI

```bash
sudo ./easyrsa init-pki
```

Creates the directory structure under `./pki/`.

---

## Build the Certificate Authority

```bash
sudo ./easyrsa build-ca nopass
```

Prompts for a Common Name. Use: `TechSecure-CA`

Output: `pki/ca.crt`

---

## Server Certificate

```bash
# Generate the request
sudo ./easyrsa gen-req server nopass

# Sign it as a server certificate
sudo ./easyrsa sign-req server server
```

Output: `pki/issued/server.crt`, `pki/private/server.key`

---

## Diffie-Hellman Parameters

```bash
sudo ./easyrsa gen-dh
```

Output: `pki/dh.pem`

Uses a 2048-bit safe prime.

---

## Client Certificate

```bash
# Generate the request (replace <username> with the actual CN)
sudo ./easyrsa gen-req <username> nopass

# Sign as client certificate
sudo ./easyrsa sign-req client <username>
```

Output: `pki/issued/<username>.crt`, `pki/private/<username>.key`

---

## Revoke a Certificate

```bash
sudo ./easyrsa revoke <username>
sudo ./easyrsa gen-crl
sudo cp pki/crl.pem /etc/openvpn/
```

Add `crl-verify /etc/openvpn/crl.pem` to `server.conf` to enforce revocation.

---

## Copy Files to OpenVPN

```bash
# Server files
sudo cp pki/ca.crt pki/private/server.key pki/issued/server.crt pki/dh.pem /etc/openvpn/

# Client files (served via HTTP or secure transfer)
sudo mkdir -p /etc/openvpn/client/keys
sudo cp pki/ca.crt /etc/openvpn/client/keys/
sudo cp pki/issued/<username>.crt /etc/openvpn/client/keys/
sudo cp pki/private/<username>.key /etc/openvpn/client/keys/
```

---

## Certificate Validity

All certificates generated by `./easyrsa sign-req` are valid for **825 days** by default (configurable in `pki/vars`).
